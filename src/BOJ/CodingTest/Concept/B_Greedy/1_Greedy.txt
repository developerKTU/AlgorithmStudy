

※ 그리디 알고리즘
    1) 개요
        - 현재 상태에서 보는 선택지 중 최선의 선택지가 전체 선택지 중 최선의 선택지라고 가정하는 알고리즘
        - 항상 최적의 해를 보장하지 않는다. (코딩테스트에서 꽤나 크리티컬한 특징임)

    2) 핵심이론
        - 해 선택 : 현재 상태에서 가장 최선이라고 생각되는 해를 선택
        - 적절성 검사 : 현재 선택한 해가 전체 문제의 제약 조건에 벗어나지 않는지 검사한다.
        - 해 검사 : 현재까지 선택한 해 집합이 전체 문제를 해결할 수 있는지 검사. 전체 문제를 해결하지 못한다면,
                   맨 위 단계로 돌아가 같은 과정을 반복한다.

    3) 주의할 점
        - 그리디 알고리즘은 잘 따져보지 않으면 반례가 생길 수 있음
            -> 그리디 알고리즘은 항상 최적의 해를 보장하지 않으므로
        - 그리디 알고리즘 문제 판단 여부는 문제의 조건을 잘 봐야함!
            -> 금액 배열에서 A(i)는 A(i-1)의 배수로 주어짐 <- 이 조건은 그리디 알고리즘에서 반례가 나올 수 없음.
               ex) 나눠야 하는 동전의 종류가 1, 5, 10, 50, 100, 1000 ... 이런식으로 주어짐
                   1, 5, 11, 41, 77 ... 이럴 경우 그리디 알고리즘은 반례가 생길 수 있음
